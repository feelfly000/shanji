# 山迹俱乐部活动审核与发布SOP

## 目标

把各户外俱乐部每周发布的组织线路集中呈现，让用户能比较时间、强度、费用、集合点和报名状态，同时控制过期信息、虚假信息和安全责任风险。

## 基本原则

- 山迹只做信息聚合，不直接组织活动。
- 山迹不代收费用。
- 报名跳转到俱乐部官方发布页或官方联系方式。
- 活动信息必须经过人工审核后展示。
- 过期活动要自动隐藏或归档。
- 高风险路线必须醒目标注。

## 每周流程

### 周一到周三：收集活动

来源可以包括：

- 合作俱乐部按模板提交。
- 俱乐部公众号公开推文。
- 俱乐部官方小程序或报名页。
- 俱乐部授权转发的信息。

不建议收录：

- 来源不明的微信群截图。
- 没有组织方名称的活动。
- 没有集合点、费用或风险说明的活动。
- 明显高风险但没有领队和安全说明的活动。

### 周三到周四：审核活动

审核字段：

- 活动标题是否清楚。
- 俱乐部名称是否明确。
- 活动日期是否准确。
- 报名截止是否明确。
- 费用是否清楚。
- 集合点是否清楚。
- 难度是否合理。
- 是否有关联路线。
- 风险提示是否具体。
- 报名链接是否来自俱乐部官方渠道。

审核状态：

| 状态 | 含义 |
|---|---|
| 待审核 | 已收集但未确认 |
| 已审核 | 可展示 |
| 需补充 | 缺少关键信息 |
| 已过期 | 活动日期已过 |
| 暂不展示 | 风险或信息不符合收录标准 |

### 周四到周五：发布周末活动

首页优先展示：

- 本周末活动。
- 报名中活动。
- 新手友好活动。
- 看海、避暑、亲子、进阶等主题活动。

### 活动结束后：归档

活动结束后：

- 从“本周活动”隐藏。
- 可进入“历史活动”。
- 如果路线价值高，可沉淀到路线库候选清单。

## 活动卡片展示规则

必须显示：

- 活动日期。
- 活动标题。
- 俱乐部名称。
- 区域。
- 难度。
- 距离。
- 集合方式。
- 费用。
- 报名状态。
- 风险提示。

建议显示：

- 关联路线。
- 费用包含。
- 人数上限。
- 装备要求。
- 报名截止。

## 报名状态规则

| 状态 | 展示建议 |
|---|---|
| 报名中 | 正常展示 |
| 余位紧张 | 高亮提醒 |
| 已满员 | 可展示但按钮置灰 |
| 已截止 | 隐藏或置灰 |
| 待确认 | 仅内部预览或弱提示 |
| 已取消 | 不展示在默认列表 |

## 免责声明建议

活动页底部建议固定显示：

```text
山迹仅聚合户外俱乐部公开或授权发布的活动信息，不直接组织活动、不代收费用。报名、行程变更、保险、领队安排和活动安全责任以活动组织方说明为准。出发前请再次确认天气、路况和报名状态。
```

## 第一版执行方式

先不做后台系统。

建议先用：

- `shanji_club_activity_template.csv` 维护活动。
- 人工审核后导出为 `shanji_club_activities_seed.json`。
- 网站展示未来7-14天活动。
- 活动过期后手动或自动隐藏。

成功标志：

用户打开“本周俱乐部活动”后，可以快速知道这周有哪些队伍、去哪、难不难、多少钱、是否还能报名。
