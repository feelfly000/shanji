# 福州徒步路线复核与采集流程

## 目的

网站要长期可信，关键不是一次性写满100条路线，而是每条路线都能说明“信息从哪里来、什么时候更新、哪些风险已确认、哪些仍需用户谨慎”。

这份流程用于把候选路线变成可发布路线。

## 路线状态

每条路线建议使用这5种状态：

| 状态 | 含义 | 是否可公开 |
|---|---|---|
| 候选待复核 | 已列入路线池，但信息来自公开资料或经验整理 | 不建议正式发布 |
| 资料已整理 | 起终点、距离、交通、风险已有初稿 | 可以内部预览 |
| 已实地复核 | 近期有人走过或有可靠轨迹/照片佐证 | 可以发布 |
| 需更新 | 可能有封路、施工、杂草、塌方、交通变化 | 暂停推荐或加醒目提示 |
| 暂不推荐 | 风险高、信息不清、路线不适合普通用户 | 不进入默认列表 |

## 发布前必查清单

### 1. 基础信息

- 路线名称是否清楚。
- 所属区域是否准确。
- 起点是否能导航到。
- 终点是否能导航到。
- 路线类型是否明确：环线、折返、穿越、串联。
- 是否存在多个版本，是否要拆成不同路线。

成功标志：

用户看到路线名和起终点后，不需要再猜“到底从哪里开始走”。

### 2. 强度参数

- 距离。
- 预计用时。
- 累计爬升。
- 最高海拔。
- 路面类型：石阶、土路、林道、公路、野路、栈道。
- 是否有长距离暴晒。
- 是否有连续陡坡。

成功标志：

用户能判断自己是否走得完，而不是只被风景吸引。

### 3. 交通与返程

- 公交/地铁是否可达。
- 自驾停车是否方便。
- 打车是否容易。
- 终点是否容易返程。
- 是否需要包车、拼车或接驳。
- 是否有末班车风险。

成功标志：

用户能知道“怎么去、怎么回”，尤其是穿越线不能只写起点。

### 4. 安全风险

- 雨后是否湿滑。
- 是否有溪谷涨水风险。
- 是否有临崖、临海、落石风险。
- 岔路是否多。
- 手机信号是否稳定。
- 是否不建议独行。
- 是否适合夜走。
- 是否适合低龄儿童或老人。

成功标志：

风险提示具体到场景，例如“雨后石阶湿滑”，而不是只写“注意安全”。

### 5. 补给和公共设施

- 起点是否有厕所。
- 途中是否有厕所。
- 终点是否有厕所。
- 途中是否有补给。
- 是否需要自带饮水。
- 是否有避雨点。

成功标志：

亲子、新手和外地游客能提前做准备。

### 6. 图片与轨迹

- 是否有封面图。
- 是否有关键路口照片。
- 是否有观景点照片。
- 是否有GPX轨迹。
- 是否有地图导航链接。
- 是否有爬升图。

成功标志：

用户能在出发前对路线环境和关键节点有直观印象。

## 路线评级建议

### 轻松

适合城市步道、景区步道、半日短线。路况清楚，出口较多，下撤容易。

### 中等

有明显爬升或距离，需要一定体力。适合普通运动者或新手进阶。

### 较难

距离较长、爬升较高、岔路较多、返程不便或补给少。不建议新手独行。

### 挑战

需要经验、轨迹、装备和结伴。第一版网站不建议把这类路线放在首页推荐。

## 更新频率

| 路线类型 | 建议复核频率 |
|---|---|
| 市区步道 | 每6个月 |
| 景区步道 | 每3-6个月，节假日前后重点检查 |
| 古道/野路 | 每3个月 |
| 溪谷路线 | 雨季前后重点检查 |
| 海岸路线 | 台风季、潮汐风险需重点提示 |
| 进阶穿越线 | 每1-3个月或收到反馈后立即复核 |

## 发现问题时怎么处理

如果出现封路、塌方、施工、落石、涨水、杂草严重、路线被围挡等情况：

1. 立刻把路线状态改为“需更新”或“暂不推荐”。
2. 在路线卡片上显示醒目提醒。
3. 详情页顶部说明问题和发现日期。
4. 如有替代路线，在专题页里推荐替代选择。
5. 待再次复核后再恢复推荐。
