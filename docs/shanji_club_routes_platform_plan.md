# 山迹：俱乐部每周线路信息共享平台方案

## 新增定位

山迹后期可以从“福州徒步线路库”升级为：

> 福州周边徒步路线库 + 户外俱乐部每周活动线路信息聚合平台。

用户可以有两种选择：

1. 自己选路线：看路线资料、风险提示、GPX轨迹，自己安排出行。
2. 报名跟队：查看俱乐部本周组织的线路，选择适合自己的活动。

## 要解决的痛点

现在很多户外俱乐部会在微信群、公众号、小红书或朋友圈发布活动，但信息分散：

- 用户不知道本周有哪些线路。
- 不同俱乐部的强度、时间、费用、集合点不好比较。
- 新手不知道哪条适合自己。
- 俱乐部发布的信息容易过期。
- 好路线和好活动沉在各自私域里，无法统一呈现。

山迹可以做一个统一的“活动线路信息板”。

## 页面入口建议

首页增加两个入口：

### 自己选路线

适合：

- 想自由安排时间的人。
- 有经验的徒步者。
- 想下载GPX自己走的人。
- 想先了解路线特点的人。

展示内容：

- 路线库。
- 搜索筛选。
- 路线详情。
- 风险提示。
- GPX下载。

### 本周俱乐部活动

适合：

- 新手。
- 想跟队的人。
- 没有车或想拼车的人。
- 想认识同伴的人。
- 不想自己研究路线的人。

展示内容：

- 活动日期。
- 俱乐部名称。
- 活动路线。
- 难度。
- 距离/爬升。
- 集合地点。
- 费用区间。
- 报名状态。
- 报名链接或联系方式。
- 风险提示。

## 活动信息字段

建议每条俱乐部活动记录包含：

| 字段 | 说明 |
|---|---|
| 活动ID | 唯一编号 |
| 活动标题 | 例如“周六奇达旗冠顶看海线” |
| 俱乐部名称 | 发布活动的组织方 |
| 活动日期 | 具体日期 |
| 报名截止 | 截止时间 |
| 路线ID | 如果对应山迹路线库，关联路线ID |
| 活动区域 | 福州、平潭、连江、永泰等 |
| 难度 | 轻松/中等/较难/挑战 |
| 距离 | 公里 |
| 爬升 | 米 |
| 集合地点 | 例如五一广场、地铁站、指定停车场 |
| 交通方式 | 大巴、自驾拼车、公共交通 |
| 费用 | 价格或区间 |
| 费用包含 | 车费、保险、领队、门票等 |
| 费用不含 | 餐食、个人装备等 |
| 人数上限 | 可选 |
| 报名状态 | 报名中、即将满员、已满员、已截止、取消 |
| 报名链接 | 跳转到俱乐部官方发布页 |
| 联系方式 | 可选，谨慎展示 |
| 装备要求 | 鞋、水、头灯、雨具等 |
| 风险提示 | 雨天、暴晒、路况、强度 |
| 信息来源 | 公众号、群公告、俱乐部提交、人工录入 |
| 更新时间 | 信息最后更新时间 |
| 审核状态 | 待审核、已发布、需更新、已过期 |

## 推荐展示方式

活动列表卡片建议这样写：

```text
周六｜奇达旗冠顶看海线
俱乐部：某某户外｜难度：中等｜约6.8km｜爬升430m
集合：福州市区指定点｜费用：128元｜状态：报名中
提醒：临海观景注意安全，台阶强度不低
```

## 和路线库的关系

俱乐部活动最好关联到路线库：

- 如果活动路线已经在山迹路线库中存在，活动卡片可以链接到路线详情。
- 如果活动路线还没有路线详情，先作为活动信息展示，后续再沉淀为路线库内容。

这样可以形成内容循环：

俱乐部活动发布 → 用户关注 → 路线被验证 → 沉淀到路线库。

## 信息来源与审核

虽然山迹不开放普通用户投稿，但可以建立“合作俱乐部信息提交”机制。

建议方式：

1. 俱乐部通过固定表格提交每周活动。
2. 山迹人工审核后发布。
3. 活动过期后自动归档或隐藏。
4. 只保留摘要，报名跳转到俱乐部自己的官方报名页。

这样既能聚合信息，又不会变成无审核的开放投稿平台。

## 风险边界

正式版需要明确：

- 山迹只做信息聚合，不直接组织活动。
- 山迹不代收费用。
- 报名、保险、领队、安全责任由活动组织方负责。
- 山迹会尽量审核信息，但活动临时变更以俱乐部官方发布为准。
- 高风险路线必须有更醒目的提示。

## 页面结构建议

### 活动首页

- 本周活动。
- 周末活动。
- 新手友好。
- 看海线路。
- 进阶线路。
- 亲子/轻徒步。
- 即将截止。

### 活动详情页

- 活动标题。
- 俱乐部名称。
- 活动时间。
- 集合点。
- 路线摘要。
- 强度参数。
- 费用说明。
- 装备要求。
- 风险提示。
- 报名状态。
- 跳转报名按钮。
- 对应山迹路线详情。

### 俱乐部页

- 俱乐部简介。
- 历史活动。
- 本周活动。
- 常走路线。
- 联系方式或官方主页。

## 第一版实现建议

不要一开始做复杂后台。建议先这样做：

1. 用表格维护活动信息。
2. 每周人工录入或由合作俱乐部按模板提交。
3. 网站只展示本周和未来活动。
4. 过期活动自动隐藏或进入历史记录。
5. 报名按钮跳转到俱乐部官方链接。

## 成功标志

用户打开山迹后，能在30秒内判断：

- 本周有哪些活动。
- 哪些适合新手。
- 哪些看海、避暑、进阶。
- 哪个俱乐部组织。
- 费用和报名状态如何。
- 是否可以自己走同一条路线。

## 下一步建议

1. 在原型首页增加“自己选路线 / 本周俱乐部活动”入口。
2. 做一个活动信息表格模板。
3. 先模拟录入5-10条俱乐部活动。
4. 设计活动卡片和活动详情页。
5. 明确免责声明和审核机制。
6. 小范围邀请1-3个俱乐部试填活动信息。
