# 单条徒步路线详情页写作模板

这个模板用于把一条路线写成网站详情页。建议每条路线都按同一结构写，用户才容易比较。

## 页面标题

```text
路线名称｜区域｜难度｜距离｜预计用时
```

示例：

```text
鼓山登山道-涌泉寺｜晋安区｜中等｜7.5km｜约3.5小时
```

## 顶部摘要

用2-3句话回答：

- 这条路线适合谁。
- 最大亮点是什么。
- 最大注意事项是什么。

模板：

```text
这是一条适合【人群】的【路线类型】路线，主要亮点是【风景/人文/体验】。全程约【距离】公里，预计【用时】小时，累计爬升约【爬升】米。需要注意【主要风险】，建议【出行建议】。
```

## 一句话亮点

控制在30-50字，用在路线卡片里。

示例：

```text
福州市区最经典的登高古道，交通方便，人文景观密集，但台阶强度不低。
```

## 路线亮点

建议写3-5点：

- 风景亮点。
- 人文亮点。
- 交通优势。
- 适合拍照/亲子/避暑/训练的原因。

## 分段说明

按用户真实行走过程写，不要只写景点名。

模板：

```text

路况：

第三段：【节点B】到【终点】

```

## 交通方式

分别写清楚：

### 公共交通

```text
可乘坐【公交/地铁】到【站点】，下车后步行约【时间】到起点。
返程可从【站点/地点】乘坐【交通方式】离开。
```

### 自驾

```text
导航到【停车点】。停车情况：【方便/紧张/收费/节假日不建议】。
```

### 打车/接驳

```text
起点打车情况：
终点打车情况：
是否需要提前约车：
```

```text

```

## 风险提示

不要写空话。建议按场景写：

- 雨后：
- 夏季：
- 夜间：
- 亲子：
- 独行：
- 岔路：
- 手机信号：

示例：

```text
雨后石阶湿滑，建议穿防滑鞋；夏季台阶路段闷热，至少携带1升饮水；不建议夜间独自下山。
```

## 下撤点

```text
可下撤位置：
下撤后交通：
不建议继续前进的情况：
```

## 装备建议

按难度给，不要过度吓人。

轻松路线：

- 运动鞋。
- 饮水。
- 防晒。

中等路线：

- 防滑徒步鞋。
- 1-1.5升饮水。
- 轻便雨具。
- 充电宝。

较难路线：

- 轨迹导航。
- 头灯。
- 雨具。
- 充足饮水和能量补给。
- 结伴出行。

## GPX轨迹

山迹允许下载GPX轨迹，但只有完成复核的路线才开放。

```text

GPX文件：
轨迹来源：
轨迹更新时间：
下载前提醒：
```

建议下载前提醒：

```text
GPX仅作为路线参考，实际路况可能因天气、施工、封路、杂草和景区管理变化而改变。出发前请查看路线更新时间和风险提示。
```

## 发布前复核项

每条路线发布前至少确认：

- 起点是否准确。
- 终点是否准确。
- 距离是否合理。
- 用时是否适合普通人。
- 风险提示是否具体。
- 是否有最新开放状态。
- 是否有更新时间。

## SEO标题建议

```text
【路线名】徒步攻略：距离、难度、交通、风险提示
```

示例：

```text
鼓山登山道到涌泉寺徒步攻略：距离、难度、交通和雨后防滑提醒
```

## SEO描述建议

控制在80-120字：

```text
整理【路线名】徒步路线信息，包括起终点、距离、预计用时、累计爬升、交通方式、适合人群、路线亮点和风险提示，帮助你判断是否适合出行。
```
