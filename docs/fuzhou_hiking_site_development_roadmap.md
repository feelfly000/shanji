# 山迹：福州徒步线路库开发路线图

## 已确认产品决策

- 网站名字：山迹。
- 副标题：福州徒步线路库。
- 路线范围：福州周边最值得走的徒步路线，包含平潭，后期可扩展宁德边界等周边。
- 地图服务：优先考虑谷歌地图或开源地图。
- 数据维护方式：表格维护 + Markdown维护。
- 用户投稿：不开放公开投稿。
- GPX轨迹：允许下载，但需要在路线复核和轨迹清洗后开放。

注意：

谷歌地图在国内访问可能不稳定，正式上线前建议保留开源地图方案作为主方案或备用方案。开源地图可以用 Leaflet + OpenStreetMap 做第一版验证。

## 当前阶段

已经完成：

- 网站定位。
- 路线字段模板。
- 首批20条候选路线。
- 可点击HTML原型。
- 前10条路线JSON种子数据。
- 路线复核流程。

下一阶段目标：

把原型升级成可以持续维护的正式网站。

## 阶段1：内容数据定型

### 要做什么

1. 确认路线字段。
2. 确认难度体系。
3. 确认标签体系。
4. 先复核8-10条样板路线。
5. 决定哪些字段第一版必须展示，哪些放到后续。

### 建议第一版必展示字段

- 路线名称
- 区域
- 难度
- 距离
- 用时
- 爬升
- 交通方式
- 是否适合新手
- 风景标签
- 适合人群
- 一句话亮点
- 风险提示
- 更新时间

### 成功标志

10条样板路线都能完整录入，并且每条路线能被筛选、搜索、打开详情。

## 阶段2：正式网站信息架构

### 页面

第一版建议做这些页面：

1. 首页：搜索、快捷筛选、推荐路线、专题入口。
2. 路线库：全部路线、筛选、排序。
3. 路线详情页：每条路线独立页面。
4. 专题页：新手、亲子、公交可达、看海、半日、进阶。
5. 关于与安全说明：评级标准、免责声明、更新机制。

### 成功标志

用户可以通过三种路径找到路线：

- 搜索关键词。
- 使用筛选。
- 从专题页进入。

## 阶段3：技术方案选择

### 方案A：静态网站

适合：

- 先做内容展示。
- 暂时不做用户登录。
- 不做公开用户投稿。

推荐技术：

- Astro、Next.js静态导出、Nuxt静态站，或纯HTML/CSS/JS。
- 数据可以先用JSON、CSV或Markdown维护。

优点：

- 便宜。
- 速度快。
- 安全风险低。
- 容易部署到GitHub Pages、Vercel、Netlify。

缺点：

- 后台编辑不方便。
- 内容维护需要遵守表格和Markdown规范。

### 方案B：轻量CMS网站

适合：

- 后续多人维护路线。
- 需要后台编辑。
- 需要上传图片和路线文件。

可选技术：

- Strapi、Payload CMS、Directus。
- WordPress也可以，但需要更注意性能和插件维护。

优点：

- 内容维护方便。
- 可扩展图片、GPX和路线状态管理。

缺点：

- 部署和维护成本更高。
- 需要账号权限管理。

### 我的建议

先用静态网站做第一版，数据用表格和Markdown维护。等路线数量、图片和GPX文件稳定后，再考虑是否上CMS。

## 阶段4：地图能力

### 第一版地图

做起点分布即可：

- 路线起点点位。
- 按区域筛选。
- 点击点位进入路线详情。

### 第二版地图

加入完整轨迹：

- GPX轨迹。
- 爬升图。
- 下撤点。
- 补给点。
- 公交站/停车点。

### 地图服务选择

已确认方向：

- 谷歌地图。
- 开源地图。

第一版建议：

- Leaflet + OpenStreetMap。

注意：

谷歌地图通常需要开发者账号、Key、费用配额和域名限制；国内用户访问也可能不稳定。开源地图更适合先做无账号原型和早期验证。

## 阶段5：专题与搜索优化

### 第一批专题

- 福州新手徒步路线
- 福州公交/地铁可达徒步路线
- 福州亲子徒步路线
- 福州看海徒步路线
- 福州半日徒步路线
- 福州鼓山鼓岭徒步路线
- 福州进阶训练路线
- 福州夏季避暑徒步路线

### 每个专题页结构

1. 一句话说明适合谁。
2. 选择标准。
3. 推荐路线列表。
4. 风险提醒。
5. 替代选择。

### 成功标志

每个专题页都不是简单堆路线，而是能帮助用户做选择。

## 阶段6：上线准备

### 上线前检查

- 手机端是否好用。
- 筛选是否准确。
- 路线详情是否有更新时间。
- 风险提示是否醒目。
- 是否有安全说明。
- 图片是否压缩。
- 页面加载是否快。
- 路线状态是否区分“已复核”和“候选”。

### 部署建议

第一版可以选择：

- Vercel：最省心，适合现代前端。
- Netlify：适合静态站。
- GitHub Pages：免费，但功能相对简单。
- 国内云服务：访问速度更好，但可能涉及备案。

### 成功标志

得到一个公开网址，普通用户可以在手机上搜索、筛选并打开路线详情。

## 阶段7：运营机制

### 每周

- 处理用户反馈。
- 检查是否有路线封闭、施工、天气风险。
- 更新首页推荐。

### 每月

- 复核重点路线。
- 增加5-10条新路线。
- 更新1-2个专题。

### 每季度

- 检查全部高热路线。
- 调整难度评级。
- 淘汰不适合推荐的路线。

## 接下来最推荐的执行顺序

1. 把原型里的10条路线做成“已复核样板”。
2. 把单文件HTML拆成正式项目结构。
3. 做路线详情独立页面。
4. 做专题页。
5. 接入真实地图起点。
6. 扩充到30条路线。
7. 小范围给朋友试用。
8. 根据反馈扩充到100条。
