# 山迹产品决策说明

## 基本定位

网站名：山迹

副标题：福州徒步线路库

一句话定位：

> 山迹是一个聚焦福州周边最值得走的徒步路线库，帮助用户按体力、交通、风景和风险快速选择适合自己的路线。

后期扩展定位：

> 山迹可以增加“本周俱乐部活动”入口，聚合户外俱乐部每周发布的组织线路，让用户既能自己走，也能报名适合的跟队活动。

## 路线范围

第一阶段：

- 福州市区。
- 闽侯、连江、长乐、永泰、福清、罗源、闽清等福州周边。
- 平潭。

后续可扩展：

- 宁德边界及福州人常去的周边徒步目的地。

收录原则：

- 不是越多越好，而是优先收录“值得走、可判断、可复核、能说明风险”的路线。
- 候选路线可以进入内部库，但正式展示前需要完成复核。

## 地图服务

已确认方向：

- 谷歌地图。
- 开源地图。

建议执行方式：

第一版优先用开源地图方案：

- Leaflet。
- OpenStreetMap。
- 起点点位。
- 路线分布。
- 后续支持GPX轨迹展示。

谷歌地图适合后续增强：

- 地图体验成熟。
- 卫星图和地形信息更强。
- 但需要确认国内访问稳定性、开发者Key、费用配额和域名限制。

## 数据维护方式

已确认：

- 表格维护。
- Markdown维护。

建议分工：

- 表格维护路线基础字段：距离、用时、难度、区域、标签、状态、更新时间。
- Markdown维护路线详情：路线概述、分段说明、交通、风险、下撤点、装备建议。
- GPX文件单独放在轨迹目录，用路线ID命名。

示例结构：

```text
routes.xlsx
routes/
  FZ001.md
  FZ002.md
gpx/
  FZ001.gpx
  FZ002.gpx
images/
  FZ001-cover.jpg
  FZ001-01.jpg
```

## 用户投稿

已确认：

- 不开放公开用户投稿。

建议保留：

- 人工维护入口。
- 联系邮箱或微信。
- 路况反馈说明。

俱乐部活动信息不等同于公开投稿。后期可以采用“合作俱乐部提交 + 山迹人工审核”的方式，聚合每周活动线路，但不开放任何人直接发布。

这样可以避免低质量内容、广告、错误路线和安全风险直接进入网站。

## 俱乐部活动入口

后期可增加两个首页入口：

- 自己选路线：用户按路线库筛选，查看路线详情、风险提示和GPX。
- 本周俱乐部活动：用户查看各户外俱乐部本周组织的线路，比较时间、强度、费用、集合点和报名状态。

活动信息建议只展示摘要和官方报名入口，山迹不直接代收费用，也不替代活动组织方承担安全责任。

## GPX轨迹

已确认：

- 允许下载GPX轨迹。

发布规则：

- 候选路线不开放GPX下载。
- 只有“已实地复核”或“轨迹已清洗”的路线才开放下载。
- GPX下载旁边必须显示风险提示和更新时间。

建议状态：

| GPX状态 | 含义 |
|---|---|
| 无轨迹 | 暂无GPX |
| 待清洗 | 有轨迹但未检查漂移和关键节点 |
| 可下载 | 已检查，可公开下载 |
| 暂停下载 | 路线状态变化，暂不建议使用 |

## 网站第一版应体现的变化

- 顶部品牌改为“山迹”。
- 副标题显示“福州徒步线路库”。
- 首页文案强调“福州周边最值得走的路线”，不是单纯100条路线堆叠。
- 路线详情页预留GPX下载区。
- 地图页先用开源地图思路表达。
- 不做用户投稿入口。
- 路线状态清楚区分“候选待复核”和“已发布”。
